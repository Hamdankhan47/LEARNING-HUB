HTML Elements with Live Examples and Explanations

<dialog> - Modal Dialog

This is a modal dialog box.

Explanation: The <dialog> element represents a popup dialog box that can be used for modal interactions or custom alerts. Adding the open attribute makes it visible by default. Use JavaScript methods like show(), showModal(), or close() to control visibility dynamically.

  • Global Attributes: Supported (e.g., id, class, style)
  • Accessibility: Consider using aria-label for better screen reader support

<details> - Expandable Content

Click to learn more

This is hidden content revealed on click.

Explanation: The <details> element creates a disclosure widget from which the user can obtain additional information. The open attribute shows the content by default. When used without JavaScript, it's ideal for FAQs or progressive content.

  • Use CSS to style open/closed states if needed
  • Improves UX by reducing initial content clutter

<canvas> - Drawing Graphics via JavaScript

Explanation: The <canvas> element provides a drawing area in the browser that you control using JavaScript. It doesn’t render anything on its own without JS code.

  • Common Attributes: width, height, id
  • Usage: Ideal for custom visualizations, games, graphs, animations

<svg> - Scalable Vector Graphics

Explanation: The <svg> element is used for drawing 2D vector graphics. Unlike <canvas>, SVG elements are part of the DOM and can be styled with CSS or animated with SMIL or JavaScript.

  • <circle> draws a circle with center (cx, cy) and radius r
  • Other shapes: <rect>, <line>, <polygon>, etc.
  • Useful for icons, graphs, and responsive visuals
